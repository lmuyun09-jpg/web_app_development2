# 食譜收藏夾 — 產品需求文件（PRD）

## 1. 專案概述

### 背景與動機

現代人越來越注重飲食健康與烹飪技巧，但在網路上找到喜歡的食譜後，常常分散在不同平台、書籤或截圖中，難以有效管理。「食譜收藏夾」系統旨在提供一個簡單易用的平台，讓使用者能集中管理自己的食譜，包含材料清單與烹飪步驟，方便隨時查閱與使用。

### 目標用戶

- **一般大眾**：對烹飪有興趣的任何人
- 家庭主婦 / 主夫：需要管理日常餐點食譜
- 烹飪愛好者：喜歡收集與整理各種食譜
- 料理新手：需要清楚的步驟指引來學習做菜

### 核心價值主張

提供一個直覺、簡潔的食譜管理工具，讓使用者能輕鬆新增、瀏覽、編輯與搜尋自己的食譜收藏，隨時隨地快速找到想做的料理。

## 2. 功能需求

### 功能一：新增食譜

**使用者故事：** 作為一位使用者，我希望能新增一道食譜（包含名稱、描述、分類），以便將喜歡的料理記錄下來。

- 輸入食譜名稱（必填）
- 輸入食譜描述 / 簡介（選填）
- 選擇食譜分類（如：中式、西式、日式、甜點、飲品等）
- 設定烹飪時間（選填）
- 設定份量 / 幾人份（選填）

### 功能二：管理材料清單

**使用者故事：** 作為一位使用者，我希望能為每道食譜列出所需的材料與份量，以便採買時可以快速參考。

- 新增材料項目（材料名稱 + 數量 + 單位）
- 編輯已新增的材料
- 刪除不需要的材料
- 材料以清單形式呈現，方便閱讀

### 功能三：管理烹飪步驟

**使用者故事：** 作為一位使用者，我希望能為每道食譜記錄詳細的烹飪步驟，以便照著步驟操作就能完成料理。

- 新增步驟（自動編號）
- 編輯已有的步驟內容
- 刪除步驟（自動重新編號）
- 調整步驟順序（上移 / 下移）

### 功能四：瀏覽與查看食譜

**使用者故事：** 作為一位使用者，我希望能瀏覽所有收藏的食譜，並點選查看完整內容，以便快速找到今天想做的菜。

- 食譜列表頁：顯示所有食譜（名稱、分類、建立時間）
- 食譜詳情頁：顯示完整資訊（描述、材料清單、烹飪步驟）
- 依分類篩選食譜

### 功能五：編輯與刪除食譜

**使用者故事：** 作為一位使用者，我希望能修改或刪除已建立的食譜，以便保持收藏夾的內容正確且整潔。

- 編輯食譜基本資訊（名稱、描述、分類等）
- 刪除整道食譜（含所有材料與步驟）
- 刪除前顯示確認提示，避免誤刪

### 功能六：搜尋食譜

**使用者故事：** 作為一位使用者，我希望能透過關鍵字搜尋食譜，以便在收藏量很多時快速找到目標。

- 依食譜名稱搜尋
- 依材料名稱搜尋（例如搜尋「雞蛋」找出所有用到雞蛋的食譜）
- 搜尋結果即時顯示

### 功能七：食譜分類管理

**使用者故事：** 作為一位使用者，我希望能將食譜依照類別分組，以便更有條理地管理收藏。

- 系統預設分類（中式、西式、日式、韓式、東南亞、甜點、飲品、其他）
- 依分類瀏覽食譜
- 食譜列表可依分類篩選

## 3. 非功能需求

### 技術限制

- **後端框架：** Flask（Python）
- **模板引擎：** Jinja2
- **資料庫：** SQLite
- **前端：** HTML + CSS + JavaScript（不使用前端框架）
- **版本控制：** Git + GitHub

### 效能考量

- 頁面載入時間應在 2 秒內
- SQLite 適合中小規模使用，預期食譜數量在數千筆以內可正常運作
- 搜尋功能使用 SQL LIKE 查詢，對小型資料集效能足夠

### 安全考量

- 所有使用者輸入需進行消毒處理（防止 XSS 攻擊）
- SQL 查詢使用參數化查詢（防止 SQL Injection）
- 表單提交使用 CSRF Token 保護
